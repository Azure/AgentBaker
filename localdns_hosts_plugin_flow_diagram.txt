═══════════════════════════════════════════════════════════════════════════════════════
                                KEY INSIGHTS
═══════════════════════════════════════════════════════════════════════════════════════

1. BOTH PATHS CONVERGE
   Non-scriptless and scriptless paths generate the same environment variables and
   execute identical bash scripts at runtime.

2. DUAL COREFILE APPROACH
   The system generates TWO Corefiles (with and without hosts plugin) and selects at
   runtime based on actual file availability, not just the flag. This provides
   resilience against misconfiguration or old VHDs.

3. GRACEFUL DEGRADATION
   If EnableHostsPlugin=true but the hosts file never populates (old VHD, timer
   failure, etc.), the system automatically falls back to the NO_HOSTS Corefile.

4. 30-SECOND WAIT
   The select_localdns_corefile() function waits up to 30 seconds for the hosts file
   to be populated, allowing the timer to fire and aks-hosts-setup.sh to run before
   finalizing the Corefile choice.

5. PERIODIC REFRESH
   The timer runs every 15 minutes to keep DNS entries fresh, handling DNS changes
   and network transitions.

6. CLOUD-AWARE
   The aks-hosts-setup.sh script uses TARGET_CLOUD to select the appropriate set of
   FQDNs for each Azure environment (Public, China, Government, etc.).
